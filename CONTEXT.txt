── OBJETIVO ────────────────────────────────────────────────────────────────────
Agente conversacional de automatización para entorno Microsoft 365 corporativo.
El usuario ejecuta el agente desde CLI, describe una tarea en lenguaje natural, y
el agente la ejecuta de forma autónoma usando herramientas (tool-calling loop).

── STACK ────────────────────────────────────────────────────────────────────────
- Python 3.x (Windows 11)
- LLM: Gemini 2.5 Flash vía API compatible con OpenAI
  · base_url: https://generativelanguage.googleapis.com/v1beta/openai/
  · API key en variable de entorno GEMINI_API_KEY
- pywin32 (win32com): integración con Outlook Desktop via COM (sin APIs cloud)
- openpyxl + pandas: lectura de planillas Excel (NO para escritura en archivos grandes)
- zipfile + XML directo: escritura en xlsx de 14MB+/87 hojas (3x más rápido que openpyxl)
- python-dotenv: variables de entorno desde .env

── HERRAMIENTAS DISPONIBLES (TOOL_DEFINITIONS en agent.py) ──────────────────────

  EMAIL (tools/email_tools.py) — requiere Outlook Desktop abierto:
  · buscar_correos_con_planillas(limite=20)
  · buscar_correos_por_asunto(palabra_clave, limite=10)
  · descargar_adjunto_correo(entry_id, attachment_index, nombre_archivo)
  · enviar_correo(destinatario, asunto, cuerpo, archivo_adjunto=None)

  SHAREPOINT (tools/sharepoint_tools.py):
  · listar_sharepoint(subcarpeta="")
  · copiar_de_sharepoint(nombre_archivo, subcarpeta="")
  · guardar_en_sharepoint(nombre_archivo, subcarpeta_destino="")

  SERVIDOR LOCAL (tools/local_tools.py):
  · listar_servidor_local(subcarpeta="")
  · copiar_del_servidor(nombre_archivo, subcarpeta="")
  · guardar_en_servidor(nombre_archivo, subcarpeta_destino="")

  EXCEL (tools/excel_tools.py):
  · leer_planilla(nombre_archivo, hoja=None)
  · validar_planilla(nombre_archivo, columnas_requeridas=None)
  · actualizar_celda(nombre_archivo, hoja, celda, valor)
  · listar_planillas_en_trabajo()

  GESTIÓN RENTA COMERCIAL (tools/gestion_renta_tools.py):
  · crear_planilla_mes(mes_code_nuevo)
      Copia el archivo más reciente y lo renombra con el nuevo código AAMM
      en R:\...\Control de Gestión Históricos\Comercial\<AAAA>\
  · actualizar_fecha_pendientes(nombre_archivo, año, mes)
      Actualiza B2 de hoja Pendientes al primer día del mes
  · agregar_vr_bursatil_pt(nombre_archivo, año, mes, precio_cuota)
      Agrega fila VR Bursátil mensual en A&R PT
  · agregar_vr_bursatil_rentas(nombre_archivo, año, mes, precio_a, precio_c, precio_i)
      Agrega 3 filas VR Bursátil en A&R Rentas (series A, C, I)
  · agregar_vr_contable_pt(nombre_archivo, año, mes, precio_cuota)
      Agrega fila VR Contable trimestral en A&R PT
  · agregar_vr_contable_rentas(nombre_archivo, año, mes, precio_a, precio_c, precio_i)
      Agrega 3 filas VR Contable en A&R Rentas
  · agregar_vr_contable_apoquindo(nombre_archivo, año, mes, precio_cuota)
      Agrega fila VR Contable en A&R Apoquindo
  · info_siguiente_accion(nombre_archivo)
      Muestra última fecha, último detalle y próxima fila disponible por hoja

  PRECIOS BURSÁTILES (tools/web_bursatil_tools.py):
  · obtener_precio_cuota(nemotecnico, año, mes)
      Intenta CMF Chile primero, luego Larraín Vial; instrucciones manuales si falla
  · obtener_precios_mes(año, mes)
      Obtiene los 4 nemotécnicos: CFITRIPT-E, CFITOERI1A/C/I

── FLUJO MENSUAL CONTROL DE GESTIÓN RENTA COMERCIAL ────────────────────────────
1. crear_planilla_mes("2604") → copia desde mes anterior
2. Copiar al WORK_DIR (copiar_del_servidor o listar_servidor_local)
3. actualizar_fecha_pendientes("2604...", 2026, 4) → B2 = 01/04/2026
4. obtener_precios_mes(2026, 3) → precios del último día del mes anterior
5. agregar_vr_bursatil_pt(...)  → A&R PT (mensual)
6. agregar_vr_bursatil_rentas(...)  → A&R Rentas series A/C/I (mensual)
   (A&R Apoquindo no tiene VR Bursátil)
7. Si es fin de trimestre (marzo, junio, septiembre, diciembre):
   · agregar_vr_contable_pt(...)
   · agregar_vr_contable_rentas(...)
   · agregar_vr_contable_apoquindo(...)
8. Guardar en servidor: guardar_en_servidor(...)

── ARQUITECTURA CLAVE: XML DIRECTO EN XLSX ──────────────────────────────────────
El xlsx es un ZIP. Solo se leen/modifican los archivos internos necesarios:
  - xl/worksheets/sheet15.xml  (A&R Apoquindo)
  - xl/worksheets/sheet16.xml  (A&R PT)
  - xl/worksheets/sheet17.xml  (A&R Rentas)
  - xl/tables/table2.xml       (Tabla133 — Apoquindo)
  - xl/tables/table3.xml       (Tabla13  — PT)
  - xl/tables/table4.xml       (Tabla1   — Rentas)
  - xl/sharedStrings.xml       (strings compartidos)
  - xl/worksheets/sheet3.xml   (Pendientes)

SHEET_CFG define: sheet_file, table_file, tabla (nombre), date_col, series, cuotas,
has_bursatil, nemotecnico/nemotecnicos.

── DETALLES CRÍTICOS DE IMPLEMENTACIÓN ──────────────────────────────────────────
1. FORMATO DE CELDAS OOXML:
   - Self-closing: <c r="D189" s="1622"/>  (sin valor)
   - Con valor:    <c r="D189" s="1622"><v>46112</v></c>
   - Con fórmula:  <c r="A189" s="106"><f>+YEAR(...)</f><v>2026</v></c>
   - String compartido: <c r="E189" s="133" t="s"><v>821</v></c>
   - NUNCA usar regex [^>]* para parsear celdas — falla con self-closing (/> contiene /)
   - Usar _find_cell_bounds() y _cell_has_value() que escanean char-by-char

2. FUNCIONES HELPER CRÍTICAS (string-scan, no regex):
   _cell_has_value(sheet_xml, ref) → True/False/None
   _find_cell_bounds(row_xml, ref) → (start, end)
   _replace_or_insert_cell(row_xml, ref, new_cell) → row_xml modificado

3. FILAS PRE-ASIGNADAS:
   Las 3 tablas tienen filas vacías pre-asignadas con estilos y fórmulas N-Y
   (Libro/Bolsa) ya presentes. Solo rellenar A-M.

4. FÓRMULAS COMPARTIDAS (A&R Rentas):
   Columna C usa <f t="shared" ref="C590:C621" si="127">. No sobreescribir si ya existe.

5. SERIAL DE FECHA EXCEL:
   date_serial = (fecha - date(1899,12,30)).days
   31/03/2026 = 46112, 01/04/2026 = 46113

6. COLUMNAS POR HOJA:
   A=YEAR, B=MONTH, C=ID, D=Fecha/SF, E=Detalle, F=Serie, G=Tipo,
   H=Monto$, I=Precio/cuota, J=Cuotas, K=UF, L=MontoUF, M=MontoUF/cuota,
   N-Y=Libro/Bolsa (pre-asignadas)

── NEMOTÉCNICOS ─────────────────────────────────────────────────────────────────
CFITRIPT-E  → A&R PT
CFITOERI1A  → A&R Rentas Serie A
CFITOERI1C  → A&R Rentas Serie C
CFITOERI1I  → A&R Rentas Serie I

── COMPATIBILIDAD MULTIPLATAFORMA ───────────────────────────────────────────────
- email_tools.py: solo funciona en Windows (pywin32/COM). En Mac retorna mensaje de error
  claro en lugar de crashear. El módulo importa sin errores en cualquier plataforma.
- Resto del agente (Excel, SharePoint, gestion_renta, web_bursatil): 100% cross-platform.
- requirements.txt: pywin32 solo se instala en Windows (marcador sys_platform=="win32").
- .gitignore y .env.example incluidos para onboarding en Mac.

── LIMITACIONES CONOCIDAS ───────────────────────────────────────────────────────
- Email (Outlook) solo en Windows
- Outlook debe estar abierto para las herramientas de email
- SHAREPOINT_DIR y LOCAL_FILES_DIR deben estar configurados en .env
- Solo lee/lista .xlsx y .xls
- leer_planilla muestra solo las primeras 20 filas
- actualizar_celda solo modifica una celda a la vez
- CMF Chile API: el endpoint puede no ser exacto — verificar con prueba real
- Los precios de Larraín Vial via scraping pueden fallar si cambia la API interna
